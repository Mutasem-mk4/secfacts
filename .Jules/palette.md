## 2026-05-14 - Fix ANSI color terminal misalignment
**Learning:** When using Go's `text/tabwriter` to format tabular data that includes ANSI escape sequences (e.g., for colored text), the tabwriter calculates column widths by counting the escape sequences as printable characters, causing severe misalignment.
**Action:** Use `tabwriter.StripEscape` when initializing the tabwriter and wrap the literal ANSI escape sequences in `\xff` bytes (e.g. `\xff\x1b[31m\xff`). The tabwriter correctly strips the `\xff` wrapper and ignores the enclosed characters during width calculations, preserving alignment while still rendering colors in the terminal. Ensure the `\xff` wrapped characters are applied conditionally, only when outputting to a terminal, to prevent invalid bytes leaking into non-terminal outputs.
