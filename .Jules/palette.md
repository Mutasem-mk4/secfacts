## 2024-04-17 - Text/Tabwriter Column Misalignment with ANSI Color Codes
**Learning:** Go's `text/tabwriter` package measures string length based on bytes. When ANSI color escape codes are injected directly into strings printed to a tabwriter, the tabwriter incorrectly includes the non-visible escape codes in its column width calculations. This results in misaligned columns when outputting text to the terminal.
**Action:** When formatting terminal output with Go's `text/tabwriter` and ANSI color codes, the `tabwriter.StripEscape` flag must be passed to `tabwriter.NewWriter`, and all ANSI escape sequences must be wrapped in `\xff` (the escape character). This allows the tabwriter to strip the `\xff` characters and ignore the enclosed bytes when calculating column widths, while still passing the ANSI codes to the final output.
