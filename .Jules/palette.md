## 2025-04-29 - Terminal Table Alignment with ANSI Colors
**Learning:** When using Go's `text/tabwriter` to render terminal tables, passing strings containing ANSI color sequences causes severe misalignment. This happens because `tabwriter` incorrectly counts the invisible ANSI escape bytes towards the column width, pushing subsequent columns out of place.
**Action:** Always instantiate `tabwriter.NewWriter` with the `tabwriter.StripEscape` flag (value `1024` or `512` based on version/Go constants, use `tabwriter.StripEscape`). Then, manually wrap any formatted text containing ANSI sequences with the `\xff` byte before and after each sequence (e.g., `\xff\033[31m\xffRed Text\xff\033[0m\xff`). This tells the tabwriter to ignore the wrapped bytes when calculating cell widths, preserving perfect table alignment while keeping the terminal colors intact.
