## 2024-05-19 - Optimize loop iteration in policy evaluation
**Learning:** Large structs like `evidence.Finding` inside `internal/domain/policy/service.go` incur significant memory allocation overhead during slice iteration via `for _, item := range list`. The cost comes from repeatedly copying the large struct's value instead of operating via its index. Additionally, storing `evidence.Finding` as the value in maps (like `seenBaseline`) requires extensive copying when map elements are created or updated.
**Action:** Replace `for _, value := range ...` with `for i := range ...` (index-based pointer semantics) during slice iteration. Store the index (an `int`) in map references (`map[Key]int`) instead of copying the full struct object into memory `map[Key]Struct`. This zero-allocation pattern reduces heap overhead significantly without affecting application logic.
