## 2026-05-07 - Avoid Massive Memory Allocation Inside Loop
**Learning:** In Go, when aggregating findings into structs containing large embedded or referenced fields (like `evidence.RootCauseCluster` storing a `Representative` of type `evidence.Finding`), repeatedly copying or assigning these large structs during slice iteration and initialization inside a tight loop causes massive unnecessary memory reallocations, leading to high CPU and GC overhead. `correlateCompact` was repeatedly overwriting the `Representative` struct by copying it on every finding iteration.
**Action:** Move the assignment of the large struct reference to the initialization phase (e.g., when appending the new cluster to the slice) and avoid redundant value copying inside the core loop. Pre-allocate slices/maps exactly when known, such as leveraging `len(representatives)` to correctly size cluster aggregations. Iterate by reference (e.g., `item := &compact[i]`) over value semantics (`for _, item := range`) to prevent copies.
