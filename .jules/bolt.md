## 2026-05-06 - Memory Optimization for RootCauseCluster Allocation
**Learning:** Allocating `clusters := make([]evidence.RootCauseCluster, 0, initialCapacity)` in `correlateCompact` used excessive memory because `evidence.RootCauseCluster` is a large struct (contains a full `Finding`), and the `initialCapacity` was vastly overestimated by using `len(compact)/4+1`. When many items map to the same cluster ID, the unique clusters are much fewer, but we allocate based on the input slice size. More importantly, we allocated slice capacity but dynamically added things via append, causing over-allocation.
**Action:** By observing that the actual number of unique clusters exactly matches `len(representatives)`, we can allocate the maps and slices with the exact capacity required (`exactCapacity := len(representatives)`). This prevents massive slice reallocation and over-allocation, and drastically cuts memory footprint (reduced over 300MB/op allocs during benchmark).
