## 2024-04-26 - Tabwriter ANSI Color Alignment Miscalculation
**Learning:** Go's `text/tabwriter` package measures cell width based on the raw byte length of strings. When ANSI escape sequences (like colors or bold text) are used inside strings passed to `tabwriter`, it counts the invisible ANSI bytes as visible characters, causing columns to misalign (become shorter than they should be) when printed to the terminal.
**Action:** When using ANSI escape codes with `tabwriter`, always initialize it with the `tabwriter.StripEscape` flag (e.g. `tabwriter.NewWriter(out, 0, 0, 2, ' ', tabwriter.StripEscape)`). Then, strictly wrap any ANSI color/styling constants inside `\xff` characters (e.g., `fmt.Fprintf(tw, "\xff%s\xffHello\xff%s\xff", colorRed, colorReset)`). This tells `tabwriter` to ignore the wrapped bytes when calculating padding and width.
